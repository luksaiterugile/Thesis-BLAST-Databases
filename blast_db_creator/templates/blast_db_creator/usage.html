Usage

There are some parameters that must be provided in order to create your own public BLAST database, some of which may be required or optional. These parameters are listed below, along with an explanation of each.

Required parameters:
Field name Description
Insert a file, File type Inserting a file from which a BLAST database will be constructed. The file that is being provided must be in FASTA format. Each sequence in FASTA format begins with a single-line header (definition line), which is followed by a line containing the sequence's data. The definition line must start with a carat (">") and then a specific sequence identifier. Blank lines in sequence are not permitted.
For more information on FASTA format, visit [3] , [4] , [5] .
Other file formats are currently unavailable.
File type default = 'Fasta'.
Database type Choose either a nucleotide database or a protein database based on the biological sequences contained in the inserted file.
Default = 'Nucleotide'.
Name of database The name of the BLAST database, which will be used to name the output files, must be inserted. The only characters that can be used in a BLAST database name are letters, digits, underscores, and hyphens.

Optional parameters:
Field name Description
Title of database Only letters, numbers, underscores, and hyphens are permitted in a database title. If title of database is not provided, name of database is used. Results of alignments will display your provided database title.
Enable sequence ID parsing If field is checked, sequence id(s) are parsed.
Create hash values Creates an index of sequence hash values if the field is checked. Hash values can be used to rapidly find out whether a given sequence is present in the BLAST database.
Version of BLAST database to be created Choose between the BLAST database versions 4 or 5. Versions 4 and 5 differ in that version 5 is taxonomy aware.
Default version = '4'.
Taxonomy ID Specify a taxonomy ID, it will be used to identify all of the sequences. Only use this field, if file that you have inserted, contains sequences that belong to the same taxonomy ID. Numeric value must be larger or equal to 0.
Taxonomy map Insert a file that contains taxonomy map. File must contain two columns, where sequence ID is mapped to the taxonomy ID.

More information on these arguments can be found at: [2]

You can view public BLAST databases that have previously been created. Users can browse all available public BLAST databases, sort databases according to a particular parameter, search for desired BLAST databases, download BLAST databases as zip file, and perform similar sequence searches using various alignment types.
Available similar sequence search types:

  • Blastn - searches in the nucleotide BLAST database using a nucleotide query.
  • Blastp - searches in the protein BLAST database using a protein query.
  • tBlastn - searches in the translated nucleotide BLAST database using a protein query.
  • tBlastx - searches in the translated nucleotide BLAST database using a translated nucleotide query.
  • Blastx - searches in the protein BLAST database using a translated nucleotide query.

Private BLAST databases are only available for registered and logged users. Logged users can choose between creating public or private BLAST databases.
The other parameters are the same as those that are used to build public BLAST databases.

Only logged user can view their created private BLAST databases.
Click here to view your private BLAST databases (link is available for logged in users only).

Only private BLAST databases can be removed. Private BLAST databases can be removed here, by selecting checkbox asocciated with BLAST database(s) and by clicking on "Delete selected BLAST databases" button. Private BLAST databases will be permanently removed.

The BLASTn program for similar sequence search requires the input of a number of parameters, some of which may be necessary or optional. Below is a list of these parameters along with an explanation of each.

Required parameters:
Field name Description
Insert nucleotide query file Inserted query file must be in FASTA format. Each sequence in FASTA format begins with a single-line header (definition line), which is followed by a line containing the sequence's data. The definition line must start with a carat (">") and then a specific sequence identifier. Blank lines in sequence are not permitted.
For more information on FASTA format, visit [3] , [4] , [5] .
Insert output filename Provide output filename to save nucleotide alignment results. The only characters that can be used in output filename are letters, digits, underscores, and hyphens.

Optional parameters:
Field name Description
Location on the query sequence If you want to limit the search to only certain query sequence locations, specify start and stop positions. Format: start-stop, delimited by hyphen.
Query strand(s) to search against database Provide query strand(s) to search against database. Depending on wanted alignments you can choose between 'minus', 'plus' or 'both' strands.
Default = 'Both'.
Task to execute Choose a 'BLASTn' task to execute.
Available 'BLASTn' tasks [2] :
  • BLASTn - the traditional program used for comparing different species
  • BLASTn-short - adapted to sequences 30 or fewer nucleotides
  • DC-megaBLAST - typically applied for inter-species comparisons
  • MegaBLAST - for extremely similar sequences
Default = 'MegaBLAST'.
Expectation value (E) threshold for saving hits Provide expectation (E) value for saving hits. When searching a database of a specific size, the expectation value indicates how many hits one can "expect" to see by accident. The lower the E value is, the more significant the match.
Default = '10'.
Word size Word size of initial match may be provided. Word size must be equal to 4 or larger.
Cost to open a gap Provide the cost to open the gap. Can be any integer value.
Cost to extend a gap Provide the cost to extend the gap. Can be any integer value.
Reward for a nucleotide match Reward for nucleotide match must be larger or equal to 0.
Penalty for a nucleotide mismatch Penalty for nucleotide mismatch must be lower or equal to 0.
Show NCBI GIs in deflines Select the field "show NCBI GIs in deflines" if GenInfo Identifiers (GIs) are present in the nucleotide BLAST database and the user wants to see GIs in his/her alignment results.
Line length for formatting alignments Provide line length for formatting alignments in output. Value must be larger or equal to 1.
Default = '60'.
Sorting option for hits Alignment results may be sorted by hits. Hits can be sorted by evalue, bit score, total score, percent identity and query coverage.
Sorting option for hps Alignment results may be sorted by High-scoring Segment Pair (HSP). HSP can be sorted by evalue, HSP score, HSP query start, HSP percent identity or HSP subject start.
Apply filtering locations as soft masks Used only for finding initial matches.
Use lower case filtering in query and subject sequence(s)
Insert file of GIs to use In order to restrict search of database to list of GIs, you must provide a file, containing GIs that you would like to use in search.
Insert file of SeqIDs to use In order to restrict search of database to list of SeqID, you must provide a file, containing SeqID that you would like to use in search.
Insert file of GIs which must not be used In order to restrict search of database to everything except the specified GIs, you must provide a file, containing GIs that must not be used in search of database.
Insert file of SeqIDs which must not be used In order to restrict search of database to everything except the specified SeqIDs, you must provide a file, containing SeqIDs that must not be used in search of database.
Taxonomy IDs to use in search of database In order to restrict search of database to include only the specified taxonoomy IDs, provide a list of taxonomy IDs, delimited by comma ','.
Taxonomy IDs which must not be included in search In order to restrict search of database to everything except the specified taxonomy IDs, provide a list of taxonomy IDs, delimited by comma ','.
Insert file of taxonomy IDs to use in search of database In order to restrict search of database to include only the specified taxonoomy IDs, you must provide a file, containing taxonomy IDs that must only be used in search.
Insert file of taxonomy IDs which must not be included in search In order to restrict search of database to everything except the specified taxonomy IDs, you must provide a file, containing taxonomy IDs that must not be used in search of database.
Percent identity Find alignments which have certain percent identity. Provide value in range 0 to 100.
Percent query coverage per hsp Find alignments based on percent query coverage per hsp. Provide value in range 0 to 100.
Maximum number of HSPs per subject sequence to save for each query Provide a maximum number of HSPs per subject sequence to save for each query. Value must be larger or equal to 1.
Turn on best hit per subject sequence Retrieve only one best hit per subject sequence by selecting this field.
Maximum number of aligned sequences to keep Default: 500 aligned sequences.
Perform only ungapped alignment When this field is selected, only ungapped alignments will be present in the output file.
Parse query and subject bar delimited sequence identifiers By enabling this field, query and subjects bar delimited sequence identifiers will be provided in the output file of alignment.

Default parameters [2] :
Task/Parameter Word size Cost to open a gap Cost to extend a gap Reward for a nucleotide match Penalty for a nucleotide mismatch
BLASTn 11 5 2 2 -3
BLASTn-short 7 5 2 1 -3
MegaBLAST 28 0 2.5 1 -2
Dc-megaBLAST 11 5 2 2 -3

Supported reward/penalty values and gap costs can be found at [2, Section "BLASTN reward/penalty values"]
More information on these arguments can be found at: [2]

Private alignment results can be viewed at private BLAST results(link is available for logged in users only).

Only private alignment results can be removed. Private alignment results can be removed here. Firsly, select a button of which BLAST+ application suite alignment results you would like to view. Available BLAST+ applications are "BLASTn", "BLASTp", "BLASTx", "tBLASTx", "tBlastn". To remove alignment result, select checkbox that is asocciated with alignment result and click on "Delete selected alignments" button. Private alignment results will be permanently removed.

To retrieve batch alignment results from many BLAST databases, use "Batch alignment generation for many BLAST databases" button in this page. Then select a filter - which BLAST+ application suite program to use for batch alignment generation for many BLAST databases. Then, select two or more private BLAST databases and provide parameters which will be used for similar sequence search. All the parameters will be applied for each BLAST database similar sequence search. Results can be viewed immediately after processing or later in alignment results page.

To update certain private BLAST database click on "Update" button in this page . To update the BLAST database, user must provide input file. All the parameters for BLAST database creator remain the same as original (father) BLAST database.

Click on "Generate the BLAST DB tree" button in this page and select any private BLAST database's checkbox to retrieve BLAST database tree. If BLAST database does not have any updates (children BLAST databases), only selected BLAST database will be seen.

To update certain alignment results of private BLAST database must have updates (children BLAST databases). If the alignment result was not updated using children BLAST database, "Update alignment" button appears next to certain alignment. User must select which child BLAST database to use for alignment result update.

When user submits form for creating BLAST databases, retrieving alignments or updating, after submit user is redirected to tasks list page, where user can periodically check if the processing of their request is done. If the task complete, it will show up, providing the name of task, item name, task status, date created and completes and also a button, which is depended on task status, where user can view errors or results of the task that was requested.
